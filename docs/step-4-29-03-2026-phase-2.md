**Target:** `Sources/ScreamerApp/MenuBarView.swift`, `Sources/ScreamerApp/SettingsView.swift`

### Task A2: Global Hotkey — Start/Stop Recording
**Target:** `Sources/ScreamerApp/AppDelegate.swift`, new `Sources/ScreamerCore/HotkeyService.swift`

Add a global keyboard shortcut so the user doesn't need the menu bar popover open to dictate.

1. Create `HotkeyService` in ScreamerCore using `CGEventTap` (requires Accessibility permission)
2. Default hotkey: **⌥Space** (Option + Space)
3. On hotkey press:
   - If idle → start recording (same path as button press)
   - If recording → stop recording and begin transcription
   - If transcribing → ignore (don't interrupt)
4. Register the tap in `AppDelegate.applicationDidFinishLaunching`
5. Unregister when Accessibility permission is not granted (degrade gracefully; show a hint in MenuBarView that hotkey is inactive)
6. The `KeyboardSettingsView` stub should now show the configured hotkey and a "Change…" button (placeholder for Task A3)

**Notes:**
- `CGEventTap` requires the app to have Accessibility permission — reuse `PermissionsService`
- The tap must run on a background thread to avoid blocking the main run loop
- No custom hotkey picker yet — hardcoded ⌥Space is enough for this task

**Tests:** Not unit-testable (CGEventTap), but write a `HotkeyServiceProtocol` so MenuBarView/AppDelegate can be tested against a stub.

**Acceptance:** Pressing ⌥Space from any app starts recording. Pressing again stops and transcribes. No popover needs to be open.

### Task A3: Floating Dictation Panel
**Target:** New `Sources/ScreamerApp/DictationPanelController.swift`, `Sources/ScreamerApp/DictationPanelView.swift`

Show a non-focus-stealing floating panel when recording is active, so the user has visual feedback without the menu bar popover.

Panel behaviour:
1. Appears centered at the bottom of the screen (above Dock) when recording starts
2. Always on top — `NSWindowLevel.floating` via `NSPanel`
3. `canBecomeKey = false` — the active app keeps focus (required for paste to work)
4. Shows: animated waveform during recording → spinner during transcription → brief success/error state → auto-dismisses after 2s
5. Escape key cancels the recording (stops audio and discards the file, does not transcribe)
6. Follows the primary display (menu bar screen) — no multi-display tracking
7. Dismisses immediately if hotkey is pressed while transcribing (do nothing, panel auto-clears on completion)

**Implementation notes:**
- Use a `NSPanel` with `styleMask: [.borderless, .nonactivatingPanel]`
- The waveform can be a simple animated bar (no real audio level needed for MVP — a pulsing animation is fine)
- Wire to the same `DictationState` used by MenuBarView so both stay in sync
- DictationPanelController should be owned by AppDelegate, not MenuBarView

**Tests:** None required for the panel itself. Ensure `DictationState` shared state doesn't introduce data races (all on MainActor).

**Acceptance:** Panel appears on recording start, shows spinner during transcription, auto-dismisses. Escape cancels. The active app's focus is never stolen.

## Track B: Model Management + Parakeet

### Task B1: Model Catalog — Static Manifest in Bundle
**Target:** New `Sources/ScreamerCore/ModelCatalog.swift`, `Resources/models-manifest.json`

Define the local model catalog before adding download infrastructure.

1. Create `models-manifest.json` in a `Resources/` directory:
   - Include: `whisper-tiny`, `whisper-base`, `whisper-small`, `whisper-medium`, `whisper-large-v3`
   - Each entry: `id`, `family` (`whisper`/`parakeet`), `display_name`, `size_mb`, `languages`, `supports_live`, `supports_translation`, `download_url`, `sha256_checksum`
2. Create `ModelCatalogEntry` Swift struct matching the JSON schema
3. Create `ModelCatalogService`:
   - `loadCatalog() -> [ModelCatalogEntry]` — reads the bundled manifest
   - `installedModels() -> [String]` — checks `~/Library/Application Support/Screamer/Models/` for installed model directories
   - `isInstalled(id:) -> Bool`
   - `installPath(for:) -> URL`

**Notes:**
- No network fetching, no signature verification yet — just the bundled local manifest
- Models directory: `~/Library/Application Support/Screamer/Models/{model-id}/`
- Parakeet can be in the manifest marked `"available": false` until Spike 1 is complete

**Tests:** Unit test `loadCatalog()` against a fixture JSON. Test `isInstalled` with a tmp directory.

**Acceptance:** `ModelCatalogService().loadCatalog()` returns the 5 Whisper models. `isInstalled` correctly reflects presence in the Models directory.

### Task B2: Model Download + Install
**Target:** New `Sources/ScreamerCore/ModelDownloadService.swift`

Add the ability to download and install a model from its catalog URL.

1. `ModelDownloadService`:
   - `download(entry: ModelCatalogEntry, progress: @escaping (Double) -> Void) async throws`
   - Uses `URLSessionDownloadTask` so downloads survive app suspend
   - Stores resume data to `~/Library/Application Support/Screamer/Downloads/{id}.resumedata` on cancellation
   - `resume(entry:)` — resumes from saved resume data if present
   - `cancel(id:)` — cancels and saves resume data
   - On completion: verifies SHA-256 checksum; if mismatch throws `ModelDownloadError.checksumMismatch`; if match moves file to `ModelCatalogService.installPath(for:)`

2. Wire to `ModelCatalogService.isInstalled` so it reflects the new file after download.

**Notes:**
- No Ed25519 manifest signing yet — that is Phase 5 (ship readiness)
- Download directory is separate from install directory — move atomically on completion
- One download at a time (no concurrency needed in this task)

**Tests:** Unit test the checksum verification logic. Mock `URLSession` to test error and cancellation paths.

**Acceptance:** Calling `download()` for `whisper-small` installs the model. `isInstalled("whisper-small")` returns true. Bad checksum throws.

## Notes for Codex

- **Swift version:** 6.0+ strict concurrency throughout. New types must be `Sendable` where appropriate.
- **Test framework:** Swift Testing (`import Testing`, `@Test`, `#expect`) — NOT XCTest.
- **No new Swift packages yet.** Don't add GRDB, Sparkle, or anything else until Phase 3/5.

- **One download at a time** — no queue, no concurrency. Keep it simple.
- **Worker stays at `127.0.0.1:8765`** — don't change the IPC contract.
- **Parakeet (B5) is explicitly gated** on the benchmark spike. Don't implement it speculatively.
