## Track A: LM Studio Integration

### Task A1: LMStudioClient — OpenAI-Compatible HTTP Client
**Target:** New `Sources/ScreamerCore/LMStudioClient.swift`

Create a client for LM Studio's local OpenAI-compatible API (`http://localhost:1234/v1/`).

1. Define `LMStudioConfiguration`:
   ```
   host: String (default "127.0.0.1")
   port: Int (default 1234)
   baseURL: URL (computed: http://{host}:{port}/v1)
   modelsURL: URL (computed: {baseURL}/models)
   chatCompletionsURL: URL (computed: {baseURL}/chat/completions)
   ```
2. Define request/response types:
   ```
   ChatMessage: role (String: "system" | "user" | "assistant"), content (String)
   ChatCompletionRequest: model (String), messages ([ChatMessage]), temperature (Double?), max_tokens (Int?), stream (Bool)
   ChatCompletionResponse: id (String), choices ([Choice])
   Choice: index (Int), message (ChatMessage), finish_reason (String?)
   LMStudioModel: id (String) — from GET /v1/models response
   ```
3. Create `LMStudioClient`:
   - `init(configuration:session:)` — injectable URLSession for testing
   - `fetchModels() async throws -> [LMStudioModel]` — GET `/v1/models`, returns loaded model list
   - `chatCompletion(_ request: ChatCompletionRequest) async throws -> ChatCompletionResponse` — POST `/v1/chat/completions` (non-streaming)
   - `streamChatCompletion(_ request: ChatCompletionRequest) -> AsyncThrowingStream<String, Error>` — POST with `stream: true`, parses SSE `data:` lines, yields content deltas
4. Define `LMStudioClientError`:
   - `connectionRefused` — LM Studio not running
   - `noModelsLoaded` — empty model list
   - `unexpectedStatusCode(Int)`
   - `streamParsingFailed`
5. Map connection errors (URLError codes `.cannotConnectToHost`, `.connectionRefused`, `.networkConnectionLost`) to `.connectionRefused` for clean error handling

**Notes:**
- LM Studio's API follows the OpenAI spec. The `/v1/models` endpoint returns `{"data": [{"id": "model-name", ...}]}`.
- SSE streaming format: lines starting with `data: ` containing JSON with `choices[0].delta.content`. Stream ends with `data: [DONE]`.
- Use `URLSession.bytes(for:)` for streaming — it's available on macOS 14+.
- Keep `stream: false` as default. Streaming is opt-in per call.

**Tests:** Unit test model list decoding, chat completion decoding, SSE delta parsing (with fixture data). Test error mapping for connection failures.

**Acceptance:** `LMStudioClient` can fetch models and send chat completions against a mock URLSession. SSE parser correctly yields content deltas from fixture stream data.

### Task A2: TranscriptActionService — Summarise, Translate, Q&A
**Target:** New `Sources/ScreamerCore/TranscriptActionService.swift`

Create a service that builds LLM prompts from transcript text and dispatches them to `LMStudioClient`.

1. Define `TranscriptAction` enum:
   ```
   case summarise
   case translate(targetLanguage: String)
   case askQuestion(question: String)
   case customPrompt(prompt: String)
   ```
2. Define `TranscriptActionResult`:
   ```
   action: TranscriptAction
   sourceTranscriptID: Int64
   resultText: String
   modelUsed: String
   createdAt: Date
   ```
3. Create `TranscriptActionService`:
   - `init(lmStudioClient:)` — injectable client
   - `perform(action:on transcript:modelID:) async throws -> TranscriptActionResult` — non-streaming, returns full result
   - `performStreaming(action:on transcript:modelID:) -> AsyncThrowingStream<String, Error>` — streaming, yields deltas
   - `availableModels() async throws -> [LMStudioModel]` — passthrough to client
4. System prompts per action:
   - **Summarise:** `"You are a helpful assistant. Summarise the following transcript concisely, preserving key points and any action items. Output only the summary, no preamble."`
   - **Translate:** `"You are a professional translator. Translate the following transcript into {targetLanguage}. Preserve the meaning and tone. Output only the translation, no preamble."`
   - **Q&A:** `"You are a helpful assistant. Answer the user's question based only on the transcript provided. If the answer is not in the transcript, say so. Be concise."`
   - **Custom:** `"You are a helpful assistant working with a transcript."` — user's prompt goes in the user message
5. Message construction:
   - Summarise/Translate: `[system prompt, user: "Transcript:\n\n{text}"]`
   - Q&A: `[system prompt, user: "Transcript:\n\n{text}\n\nQuestion: {question}"]`
   - Custom: `[system prompt, user: "{prompt}\n\nTranscript:\n\n{text}"]`
6. For very long transcripts (>12000 chars), prepend a note to the system prompt: `"The transcript is long. Focus on the most important content."`

**Notes:**
- Do not truncate transcripts — let the LLM handle context limits. LM Studio will return an error if the context is exceeded, which we surface to the user.
- Temperature: 0.3 for summarise/translate (deterministic), 0.7 for Q&A/custom (creative).
- max_tokens: nil (let the model decide).

**Tests:** Unit test prompt construction for each action type. Test that long transcripts get the prepended note. Test with mocked `LMStudioClient`.

**Acceptance:** Each action type produces the correct message array. Streaming passthrough works. Mocked tests pass.

## Notes for Codex

- **Swift version:** 6.0+ strict concurrency throughout. New types must be `Sendable` where appropriate.
- **Test framework:** Swift Testing (`import Testing`, `@Test`, `#expect`) — NOT XCTest.
- **No new Swift package dependencies.** LM Studio client uses Foundation `URLSession` only.
- **LM Studio API:** OpenAI-compatible at `http://localhost:1234/v1/`. Endpoints: `GET /v1/models`, `POST /v1/chat/completions`. SSE streaming via `URLSession.bytes(for:)`.
- **SSE parsing:** Lines prefixed with `data: ` contain JSON. Each chunk has `choices[0].delta.content` with the text delta. Stream ends with `data: [DONE]`.
- **Worker stays at `127.0.0.1:8765`** — LM Studio is a separate service. Don't route LLM calls through the worker.
- **Database migrations:** Use GRDB's `DatabaseMigrator.registerMigration()` — append new migrations after existing ones. Never modify the `"createTranscripts"` migration.
- **Segments JSON storage:** Encode `[TranscriptSegment]` as a JSON string in a TEXT column. Use `JSONEncoder`/`JSONDecoder` in custom `Codable` implementation.
- **Cascade delete:** Use `REFERENCES transcripts(id) ON DELETE CASCADE` and ensure `PRAGMA foreign_keys = ON` is set on each connection.
- **Streaming UI:** Use `AsyncThrowingStream` to yield SSE deltas. The UI observes the stream via a `@Published` property that appends each delta.
- **No new Python packages or worker changes.** Phase 4 is Swift-side only (plus the LM Studio external dependency).
- **Settings tab order:** General, Models, AI (new), Keyboard.
- **Action bar placement:** Inside expanded transcript row, below full text, above previous actions. Only visible when LM Studio is configured.
- **Temperature defaults:** 0.3 for summarise/translate, 0.7 for Q&A/custom. Not user-configurable yet.
