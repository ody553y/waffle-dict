Phase 3 turns Screamer from a dictation tool into a **transcription workspace**. Two parallel tracks:

**Track A — File Transcription Workspace** (new UI surface, GRDB persistence, export)

## Track A: File Transcription Workspace

### Task A1: GRDB Transcript History — Schema + Service
**Target:** New `Sources/ScreamerCore/TranscriptStore.swift`, update `Package.swift`

Add GRDB as a dependency and create the transcript persistence layer.

1. Add `GRDB` Swift package: `https://github.com/groue/GRDB.swift`, from `7.0.0`
2. Define `TranscriptRecord`:
   ```
   id: Int64 (autoincrement primary key)
   createdAt: Date
   sourceType: String ("dictation" | "file_import")
   sourceFileName: String? (original filename for imports, nil for dictation)
   modelID: String
   languageHint: String?
   durationSeconds: Double?
   text: String
   ```
3. Create `TranscriptStore`:
   - `init(databasePath:)` — opens/creates the SQLite database at `~/Library/Application Support/Screamer/transcripts.sqlite`
   - `save(_ record: TranscriptRecord) throws -> TranscriptRecord` — inserts and returns with assigned ID
   - `fetchAll(limit:offset:) throws -> [TranscriptRecord]` — newest first
   - `search(query:limit:) throws -> [TranscriptRecord]` — full-text search on `text` column using FTS5
   - `delete(id:) throws`
   - `fetchOne(id:) throws -> TranscriptRecord?`
4. Set up FTS5 virtual table on `text` for fast search
5. Database file location: `~/Library/Application Support/Screamer/transcripts.sqlite`

**Tests:** Unit test save, fetch, search, delete against an in-memory database.

**Acceptance:** `TranscriptStore` round-trips records. FTS search returns matching results. Tests pass.

## Track B: Production Hardening

### Task B1: Streaming SHA-256 Checksum for Large Models
**Target:** `Sources/ScreamerCore/ModelDownloadService.swift`

The current `checksum(for:)` loads the entire file into memory. For `whisper-large-v3` (3.1GB) this will spike memory.

1. Replace `Data(contentsOf:)` with a streaming reader:
   - Open file handle, read in 1MB chunks, feed each chunk to `SHA256.update()`
   - Return final digest
2. Keep the existing `verifyChecksum(of:expectedChecksum:)` API unchanged

**Tests:** Existing checksum tests should still pass. Add a test with a file larger than 1MB to verify streaming works.

**Acceptance:** `verifyChecksum` works on large files without loading them entirely into memory. Existing tests pass.

### Task B3: Worker — Word-Level Timestamps in Response
**Target:** `worker/screamer_worker/server.py`, `worker/screamer_worker/models.py`, `worker/screamer_worker/backends/faster_whisper.py`

Add optional word-level timestamp data to the transcription response for future SRT/VTT segmentation.

1. Extend `FileTranscriptionResponse` with an optional `segments` field:
   ```python
   @dataclass(frozen=True)
   class TranscriptionSegment:
       start: float    # seconds
       end: float      # seconds
       text: str

   @dataclass(frozen=True)
   class FileTranscriptionResponse:
       job_id: str
       backend_id: str
       text: str
       segments: list[TranscriptionSegment] | None = None
   ```
2. Update `FasterWhisperBackend.transcribe_file()`:
   - Return segments from `faster-whisper` (it already provides them)
   - Map each segment to `TranscriptionSegment(start, end, text)`
3. Update `ParakeetBackend.transcribe_file()`:
   - Return `segments=None` (NeMo doesn't provide word-level timestamps in the basic API)
4. Update the server to include `segments` in the JSON response when present
5. Update Swift `FileTranscriptionResponse` to decode `segments` as optional

**Tests:** Update the existing transcription test to verify segments are present in the response. Add a test that segments are `null` for backends that don't support them.

**Acceptance:** Whisper transcriptions include timestamped segments. Parakeet returns null segments. Swift decodes both correctly.

## Notes for Codex

- **Swift version:** 6.0+ strict concurrency throughout. New types must be `Sendable` where appropriate.
- **Test framework:** Swift Testing (`import Testing`, `@Test`, `#expect`) — NOT XCTest.
- **New dependency: GRDB only.** Add `https://github.com/groue/GRDB.swift` from `7.0.0`. No other new packages.
- **Database location:** `~/Library/Application Support/Screamer/transcripts.sqlite` — use the same `applicationSupportDirectory` pattern as `ModelCatalogService`.
- **FTS5:** GRDB supports FTS5 natively. Use `db.create(virtualTable:using:)` with `FTS5()`.
- **Worker stays at `127.0.0.1:8765`** — don't change the IPC contract.
- **Export formats:** SRT/VTT are single-segment for now (full duration). Word-level segmentation comes from B3, but A5 should work without it.
- **File import accepts:** `.wav`, `.mp3`, `.m4a`, `.flac`, `.ogg`. The worker handles format conversion internally via faster-whisper.
- **No new Python packages** unless required by B3 (faster-whisper already provides segments).
- **History window:** Standard `Window` scene in SwiftUI, not `MenuBarExtra`. Opened via button, not always visible.
- **One file transcription at a time** — same constraint as model downloads. No concurrency queue needed.
