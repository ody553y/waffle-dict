   - Menu Bar strings (~10)

## Notes for Codex

- **Swift version:** 6.0+ strict concurrency throughout. New types must be `Sendable` where appropriate.
- **Test framework:** Swift Testing (`import Testing`, `@Test`, `#expect`) — NOT XCTest.
- **Python test framework:** `unittest` — match existing worker test patterns.
- **No new Swift packages** in Phase 6. All work uses existing framework capabilities.
- **No new Python dependencies** in Phase 6. Worker is not modified.
- **Database migrations:** Append after existing migrations in `TranscriptStore`. Phase 6 adds two columns: `speakerMap TEXT` and `notes TEXT`.
- **FTS5 rebuild:** When adding `notes` to the FTS5 index, issue `INSERT INTO transcripts_fts(transcripts_fts) VALUES('rebuild')` in the migration.
- **String Catalog:** Use `.xcstrings` format. Add to WaffleApp target resources in Package.swift. Requires `defaultLocalization: "en"` on the package.
- **Accessibility:** Use SwiftUI accessibility modifiers (`.accessibilityLabel`, `.accessibilityHint`, `.accessibilityAddTraits`). For macOS-specific notifications, use `NSAccessibility.post(element:notification:)`.
- **Keyboard shortcuts:** Use `.keyboardShortcut()` modifier on Buttons and `.onKeyPress()` for custom key handling in the list view. Arrow key navigation requires `@FocusState` management.
- **Settings tab order:** General, Models, AI, Keyboard (unchanged from Phase 5).
- **Speaker map storage:** JSON-in-TEXT column, same pattern as `segments`. Encode with `JSONEncoder`, decode with `JSONDecoder`.
- **Export speakerMap threading:** Pass `TranscriptRecord.speakerMap` to `TranscriptExporter` methods. Don't store resolved names on segments.
- **Menu bar preview:** The `MenuBarExtra` uses `.window` style. The preview card is a SwiftUI view inside the existing popover, not a new window.
- **Filter state:** Keep filter state in the ViewModel, not persisted. Resets on app restart. Future phase could add saved filters.
- **Debounced notes save:** Use `Task` with `try await Task.sleep(for: .milliseconds(500))` and cancellation for debounce. Cancel previous save task on each keystroke.
