## Track A: Export Pipelines

An **export pipeline** is a named, ordered list of prompt-template steps that run automatically after each transcription (or on demand). Each step feeds the previous step's output as its input, enabling chains like "Summarise → Extract Action Items → Translate to French".

### Task A1: ExportPipeline Model and ExportPipelineStore
**Target:** `Sources/WaffleCore/ExportPipeline.swift` (new)

1. Define the data model:
   ```swift
   public struct ExportPipelineStep: Codable, Equatable, Identifiable, Sendable {
       public var id: UUID
       public var templateID: UUID        // references a PromptTemplate
       public var templateName: String    // denormalized snapshot for display when template is deleted
       public var templatePrompt: String  // denormalized snapshot — used at run time
   }

   public struct ExportPipeline: Codable, Equatable, Identifiable, Sendable {
       public var id: UUID
       public var name: String
       public var steps: [ExportPipelineStep]   // ordered; min 1, max 10
       public var runAutomatically: Bool         // if true, runs after every transcription
       public var outputDestination: ExportPipelineOutputDestination
       public var createdAt: Date
   }

   public enum ExportPipelineOutputDestination: String, Codable, CaseIterable, Sendable {
       case clipboard     // copy final step output to clipboard
       case none          // results saved as transcript actions only
   }
   ```

2. Create `ExportPipelineStore`:
   ```swift
   public final class ExportPipelineStore: @unchecked Sendable {
       public static let storageKey = "exportPipelines"
       public static let maxPipelineCount = 10

       public init(userDefaults: UserDefaults = .standard) { ... }

       public func load() -> [ExportPipeline]
       public func save(_ pipelines: [ExportPipeline])
       public func add(_ pipeline: ExportPipeline)      // enforces maxPipelineCount
       public func update(_ pipeline: ExportPipeline)
       public func delete(id: UUID)
   }
   ```
   - Stored as JSON array under `UserDefaults` key `exportPipelines`.
   - Max 10 pipelines. `add` silently drops if at cap.
   - Each pipeline stores `templatePrompt` as a snapshot so pipelines continue working after template edits or deletes.

**Notes:**
- Steps store a denormalized `templatePrompt` so the pipeline runs even if the source `PromptTemplate` is later deleted or renamed. The snapshot is taken at pipeline creation/update time.
- `runAutomatically` pipelines are triggered fire-and-forget from `DictationController` (wired in A2).
- `ExportPipelineOutputDestination.clipboard` copies the final step's output to `NSPasteboard.general`. `none` saves only to the action log.
- No new dependencies. JSON storage in UserDefaults is sufficient for ≤10 pipelines.

**Tests:** Test JSON round-trip for `ExportPipeline` and `ExportPipelineStep`. Test `add` enforces 10-pipeline cap. Test `delete` removes correct pipeline by ID. Test `update` replaces existing pipeline. Test `load` returns empty array when key is absent.

**Acceptance:** `ExportPipelineStore` can persist, load, and mutate up to 10 pipelines across app restarts.

### Task A2: ExportPipelineRunner
**Target:** `Sources/WaffleCore/ExportPipelineRunner.swift` (new)

Run a pipeline's steps sequentially, feeding each step's output as the next step's input transcript text.

1. Create `ExportPipelineRunner`:
   ```swift
   public final class ExportPipelineRunner: Sendable {
       private let actionService: TranscriptActionService
       private let transcriptStore: TranscriptStore

       public init(
           actionService: TranscriptActionService,
           transcriptStore: TranscriptStore
       ) { ... }

       /// Run all steps in the pipeline against the given transcript.
       /// Each step's output feeds the next step's input.
       /// Returns the final step output text, or throws on first step failure.
       public func run(
           pipeline: ExportPipeline,
           transcript: TranscriptRecord,
           modelID: String
       ) async throws -> String

       /// Run then apply output destination (clipboard copy, etc.).
       public func runAndDeliver(
           pipeline: ExportPipeline,
           transcript: TranscriptRecord,
           modelID: String
       ) async throws
   }
   ```

2. Implementation of `run(pipeline:transcript:modelID:)`:
   - Assert `pipeline.steps` is non-empty; throw `ExportPipelineError.emptyPipeline` if empty.
   - Start with `currentText = transcript.text`.
   - For each step in order:
     - Build a scratch `TranscriptRecord` from the running transcript but with `text = currentText` (so each step acts on accumulated output).
     - Call `actionService.perform(action: .customPrompt(prompt: step.templatePrompt), on: scratchRecord, modelID: modelID)`.
     - Save the `TranscriptActionResult` to `transcriptStore.saveAction(_:)` so the action log captures all intermediate steps.
     - Set `currentText = result.resultText` for the next step.
   - Return final `currentText`.

3. Implementation of `runAndDeliver`:
   - Call `run(...)`.
   - If `pipeline.outputDestination == .clipboard`: `NSPasteboard.general.clearContents(); NSPasteboard.general.setString(finalText, forType: .string)`.

4. Define `ExportPipelineError`:
   ```swift
   public enum ExportPipelineError: Error, Equatable, Sendable {
       case emptyPipeline
   }
   ```

5. In `DictationController.saveTranscriptToHistory`, after saving succeeds, find all pipelines with `runAutomatically == true` and fire each as a background task:
   ```swift
   let autoPipelines = pipelineStore.load().filter(\.runAutomatically)
   for pipeline in autoPipelines {
       Task.detached(priority: .utility) {
           try? await runner.runAndDeliver(pipeline: pipeline, transcript: savedRecord, modelID: activeModelID)
       }
   }
   ```
   - Fire-and-forget. Never block dictation flow. Errors silently drop.

**Notes:**
- Each intermediate step's `TranscriptActionResult` is saved with `actionType: "pipeline_step"` and the step's `templateName` embedded in the result for display in the action log.
- `NSPasteboard` is only available in `WaffleApp` module. Move clipboard delivery to `DictationController` or a thin app-layer wrapper, keeping `ExportPipelineRunner` in `WaffleCore` clipboard-free. Alternatively: `runAndDeliver` returns the final text and the caller in `DictationController` applies clipboard copy. Choose: have `run` return the result and let the caller in `WaffleApp` handle delivery. This keeps `WaffleCore` free of AppKit.
- Preferred approach: `ExportPipelineRunner.run(...)` lives in `WaffleCore` and returns `String`. A thin `func applyOutputDestination(_ text: String, destination: ExportPipelineOutputDestination)` helper lives in `WaffleApp/DictationController.swift`.

**Tests:** Test that `run` chains step outputs correctly (mock `TranscriptActionService`). Test that each step's result is saved as a `TranscriptActionResult`. Test `emptyPipeline` error. Test that `run` propagates the first step's error. Test that intermediate step output feeds the next step's prompt correctly (not the original transcript text).

**Acceptance:** Running a 3-step pipeline produces 3 saved action results. The final step's output is delivered to the configured destination. Auto-run pipelines trigger after every successful dictation.

## Track D: Statistics Dashboard

### Task D1: Statistics Queries in TranscriptStore
**Target:** `Sources/WaffleCore/TranscriptStore.swift`

1. Define `TranscriptStatistics`:
   ```swift
   public struct TranscriptStatistics: Equatable, Sendable {
       public let transcriptCount: Int
       public let totalWords: Int
       public let totalDurationSeconds: Double
       public let averageDurationSeconds: Double    // 0 if transcriptCount == 0
       public let byModel: [String: Int]            // modelID → transcript count
       public let dailyCounts: [String: Int]        // "yyyy-MM-dd" → transcript count, last 30 days
   }
   ```

2. Add `func statistics(since: Date? = nil) throws -> TranscriptStatistics` to `TranscriptStore`:
   - `transcriptCount`: `SELECT COUNT(*) FROM transcripts [WHERE createdAt >= since]`
   - `totalWords`: fetch all `text` columns, split on whitespace, count (or use SQLite `length(text) - length(replace(text, ' ', '')) + 1` approximation)
   - `totalDurationSeconds`: `SELECT COALESCE(SUM(durationSeconds), 0) FROM transcripts`
   - `byModel`: `SELECT modelID, COUNT(*) FROM transcripts GROUP BY modelID`
   - `dailyCounts`: `SELECT strftime('%Y-%m-%d', createdAt) as day, COUNT(*) FROM transcripts WHERE createdAt >= {30 days ago} GROUP BY day`
   - Run in a single GRDB `read` closure for consistency.

3. Word count implementation note: SQLite word-count approximation (`length - length(replace) + 1`) is fast but slightly inaccurate for leading/trailing spaces. For accuracy, prefer fetching `text` column values and counting in Swift with `.split(separator: " ").count`. Since this runs in a background task it's acceptable.

**Notes:**
- `since: nil` aggregates all time. `since: Date()` offset by 30 days gives monthly stats.
- `dailyCounts` always covers last 30 days regardless of `since` parameter (it's the sparkline data for the chart).
- `averageDurationSeconds` is derived: `totalDurationSeconds / max(1, transcriptCount)`.
- The stats query should run in `Task.detached(priority: .utility)` from the view layer to avoid blocking the main thread.
- `strftime('%Y-%m-%d', createdAt)` works because `createdAt` is stored as a UNIX timestamp (Double) in GRDB, and SQLite's `strftime` accepts numeric Unix timestamps when wrapped with `datetime(createdAt, 'unixepoch')`. Adjust the query: `strftime('%Y-%m-%d', datetime(createdAt, 'unixepoch'))`.

**Tests:** Test `transcriptCount` matches inserted record count. Test `totalDurationSeconds` sums correctly including nil durations (treat nil as 0). Test `byModel` groups correctly. Test `dailyCounts` covers last 30 days and excludes older records. Test `averageDurationSeconds` is 0 when `transcriptCount` is 0.

**Acceptance:** `TranscriptStore.statistics()` returns accurate aggregate data in a single DB read pass.

## Notes for Codex

- **Swift version:** 6.0+ strict concurrency. All new types must be `Sendable`.
- **Test framework:** Swift Testing (`import Testing`, `@Test`, `#expect`) — NOT XCTest.
- **No new Swift packages.** `Charts` framework is available system-side on macOS 13+ if used for D2, but a plain SwiftUI bar chart with `Canvas` or `HStack` is sufficient and preferred.
- **`ExportPipelineStore` storage key:** `exportPipelines`. Max 10 pipelines. JSON array in UserDefaults.
- **`ExportPipelineStep` snapshot:** Always copy `templatePrompt` from `PromptTemplate` at pipeline creation/edit time. Do not store only the templateID — templates can be deleted.
- **`ExportPipelineRunner` in WaffleCore:** Must NOT import AppKit or use `NSPasteboard`. Clipboard delivery belongs in `WaffleApp` (in `DictationController` or a thin helper). `ExportPipelineRunner.run(...)` returns `String`; the caller decides what to do with it.
- **Auto-run pipeline fire-and-forget:** Use `Task.detached(priority: .utility)` in `DictationController.saveTranscriptToHistory`. Errors are swallowed silently (same pattern as webhook and iCloud backup delivery).
- **`reviewStatus` column migration:** Add to the existing `DatabaseMigrator` in `TranscriptStore`. Name the migration `"addReviewStatusColumn"`. Existing records get `reviewStatus = nil` automatically via GRDB optional column behavior.
- **`ReviewStatus` constants:** `ReviewStatus.approved = "approved"`, `ReviewStatus.dismissed = "dismissed"`. Use these string constants everywhere — never hardcode the strings.
- **`[safe: index]` subscript:** Add `extension Collection { subscript(safe index: Index) -> Element? { indices.contains(index) ? self[index] : nil } }` as a private extension in `ReviewQueueView.swift` or as a `WaffleCore` utility if used in multiple places.
- **Notification name for review queue:** `Notification.Name.waffleOpenReviewQueue`. Define it as an extension on `Notification.Name` in `TranscriptArchiveImportNotification.swift` (the existing notification helper file) or a new `NotificationNames.swift` file.
- **`updateSegments` text derivation:** `segments.map(\.text).joined(separator: " ")`. This is the canonical way to regenerate full text from segments. Never let callers set `text` and `segments` independently.
- **Segment editor speaker rename:** Only updates the local segment's `speaker` field. Does NOT touch `SpeakerProfileStore`. Document this in a comment in the editor.
- **Statistics `strftime` query:** Use `datetime(createdAt, 'unixepoch')` because GRDB stores `Date` as a Unix timestamp Double. The correct SQLite expression is `strftime('%Y-%m-%d', datetime(createdAt, 'unixepoch'))`.
- **Statistics view loading:** Use `.task { }` with `Task.detached(priority: .utility)` inside to avoid blocking the main actor during DB reads.
- **`SegmentEditorSheet` performance:** For transcripts with 200+ segments, use `LazyVStack` inside `ScrollView` instead of `List` to avoid row height pre-computation overhead.
- **Review queue menu bar badge:** Read `unreviewedCount()` on `NSApplicationDidBecomeActiveNotification` and after each approve/dismiss action. Badge shows as "(N)" suffix on the menu item title. Hide the suffix (or show "Review Queue") when count is 0.
- **`⌘⇧R` shortcut for review queue:** Verify it doesn't conflict with `TranscriptHistoryView` or system shortcuts before finalising. Alternative: `⌥⌘Q`.
