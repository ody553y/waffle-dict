## Phase 10 Scope

Phase 10 is the **distribution and polish** phase, optimized for:
- **GitHub Releases + Sparkle auto-update**
- **No App Store track**

Tracks:
- **Track A — Release & Update Pipeline** (Developer ID signing, notarization, GitHub release artifacts, Sparkle feed/signatures)
- **Track B — First-Run Onboarding** (guided setup for core prerequisites)
- **Track C — Localization Hardening** (key usage coverage + completeness guardrails)
- **Track D — Accessibility & Interaction Polish** (VoiceOver, keyboard flow, large-text/clipping behavior)

Parallelism:
- A1 + B1 + C1 + D1 can start in parallel after the stabilization gate.
- Track A is highest priority because it defines shippability.

   - Fail release build if `SUPublicEDKey` equals placeholder (`REPLACE_WITH_SPARKLE_PUBLIC_ED_KEY`) or is empty.

### Task A2: Appcast Generation + Signing Integration
**Targets:**

- new script: `scripts/generate-appcast.sh` (or equivalent)

1. Add deterministic appcast generation from release inputs:
   - version

   - archive length

2. Integrate output with existing `sign_update` flow.

3. Emit a ready-to-publish appcast file suitable for GitHub Pages or equivalent static hosting.

**Tests/Checks:** script-level validation (required fields, malformed URL/version rejection), deterministic output snapshot test if feasible.

**Acceptance:** A single documented command sequence can produce signed DMG metadata + publishable appcast.

### Task A3: GitHub Release Publishing Runbook (Decision-Complete)
**Targets:**
- `scripts/README.md`
- new doc: `docs/release-runbook.md`

1. Document exact end-to-end release flow:
   - Build/sign app + DMG
   - Notarize/staple
   - Sparkle sign update archive
   - Generate appcast entry/file
   - Publish DMG to GitHub Release

2. Include required secrets/env vars and verification commands (`codesign --verify`, `spctl`, update-check sanity).

3. Define rollback procedure for bad release/appcast.

**Acceptance:** A new engineer can run the release from clean checkout without guessing any steps.
