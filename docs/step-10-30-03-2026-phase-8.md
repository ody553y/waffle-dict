## Phase 8 Scope

Phase 8 is the **automation & ecosystem** phase: exposing Waffle's core functionality to the macOS Shortcuts app via App Intents, sending webhook notifications when new transcripts arrive, and backing up transcripts to iCloud Drive.

**Track A — Shortcuts App Integration** (App Intents for Shortcuts.app, Siri, and Spotlight)
**Track B — Webhook Notifications** (configurable HTTP POST on new transcript, HMAC signing, delivery log)

Tracks A and B are fully independent and can start simultaneously. Track C requires entitlements file changes (done first in C1) before the Swift code (C2, C3).

Parallelism: A1 + B1 + C1 can all start simultaneously. Within Track A: A1 → A2 → A3 (sequential). Within Track B: B1 → B2 → B3 (sequential). Within Track C: C1 → C2 → C3 (sequential).

### Task A2: GetLastTranscriptIntent
**Target:** `Sources/WaffleApp/Intents/TranscriptIntents.swift` (new)

Expose the most recent transcript as an App Intent result so users can pipe it into other Shortcuts actions.

1. Add `GetLastTranscriptIntent`:
   ```swift
   struct GetLastTranscriptIntent: AppIntent {
       static let title: LocalizedStringResource = "Get Last Transcript"
       static let description = IntentDescription("Returns the text of the most recent Waffle transcript.")
       static let openAppWhenRun = false

       func perform() async throws -> some IntentResult & ReturnsValue<String> {
           guard let store = TranscriptIntentBridge.shared.transcriptStore else {
               throw IntentError.custom("Transcript store is unavailable.")
           }
           let records = try store.fetchAll(limit: 1)
           guard let latest = records.first else {
               throw IntentError.custom("No transcripts found.")
           }
           return .result(value: latest.text)
       }
   }
   ```
2. Create `TranscriptIntentBridge` — `@unchecked Sendable` singleton holding a weak reference to `TranscriptStore`:
   ```swift
   final class TranscriptIntentBridge: @unchecked Sendable {
       static let shared = TranscriptIntentBridge()
       weak var transcriptStore: TranscriptStore?
   }
   ```
3. In `AppDelegate` / `WaffleApp.swift`, register the store:
   ```swift
   TranscriptIntentBridge.shared.transcriptStore = transcriptStore
   ```
4. Add a Shortcut phrase suggestion in `WaffleShortcuts`:
   ```swift
   AppShortcut(intent: GetLastTranscriptIntent(), phrases: ["Get last \(.applicationName) transcript"])
   ```

**Notes:**
- `ReturnsValue<String>` makes the transcript text available as output for downstream Shortcuts actions (e.g., "Add to Note", "Send Message").
- `IntentError.custom(_:)` is the recommended way to surface user-facing errors from App Intents.
- `TranscriptStore` is `@unchecked Sendable`. The bridge can safely hold a weak reference without wrapping in an actor since it's read-only via `fetchAll`.
- The intent is synchronous from the user's perspective but calls the throwing `fetchAll` — errors become Shortcut errors shown in the Shortcuts app.

**Tests:** Test that `GetLastTranscriptIntent.perform()` returns the most recent transcript text. Test the error path when no transcripts exist. Test the error path when the store is nil.

**Acceptance:** "Get Last Transcript" appears in Shortcuts.app and returns the transcript text for use in downstream actions (e.g., "Copy to Clipboard", "Add to Note", "Send Message").

### Task A3: SearchTranscriptsIntent
**Target:** `Sources/WaffleApp/Intents/TranscriptIntents.swift`

Let users search transcripts by text from Shortcuts.app and return results.

1. Add `SearchTranscriptsIntent`:
   ```swift
   struct SearchTranscriptsIntent: AppIntent {
       static let title: LocalizedStringResource = "Search Transcripts"
       static let description = IntentDescription("Searches Waffle transcripts and returns matching text.")
       static let openAppWhenRun = false

       @Parameter(title: "Query", description: "The text to search for.")
       var query: String

       @Parameter(title: "Max Results", description: "Maximum number of results to return.", default: 5)
       var maxResults: Int

       func perform() async throws -> some IntentResult & ReturnsValue<[String]> {
           guard let store = TranscriptIntentBridge.shared.transcriptStore else {
               throw IntentError.custom("Transcript store is unavailable.")
           }
           let clamped = max(1, min(maxResults, 20))
           let records = try store.search(query: query, limit: clamped)
           let texts = records.map { $0.text }
           return .result(value: texts)
       }
   }
   ```
2. Add to `WaffleShortcuts`:
   ```swift
   AppShortcut(
       intent: SearchTranscriptsIntent(),
       phrases: ["Search \(.applicationName) for \(\.$query)"]
   )
   ```

**Notes:**
- `@Parameter` properties are filled by the user in the Shortcuts editor or via Siri voice input.
- `maxResults` defaults to 5, clamped to 1–20 to avoid flooding downstream actions.
- Returns `[String]` (array of transcript texts). The Shortcuts app can iterate over the array with "Repeat with each item".
- Uses the existing `TranscriptStore.search(query:limit:)` which uses FTS5 under the hood.
- The `@Parameter` for `query` can be provided via Siri: "Search Waffle for [query]" where `[query]` is a variable substitution.

**Tests:** Test that `perform()` returns correctly matched transcript texts. Test `maxResults` clamping to 1–20. Test empty query returns empty array (or up to maxResults recent transcripts). Test nil store throws `IntentError`.

**Acceptance:** "Search Transcripts" appears in Shortcuts.app with configurable query and result count parameters. Results can be used in downstream Shortcuts actions.

### Task B2: Webhook Delivery Service
**Target:** `Sources/WaffleCore/WebhookService.swift` (new)

Send a signed HTTP POST to the configured webhook endpoint after each successful transcription.

1. Create `WebhookPayload`:
   ```swift
   public struct WebhookPayload: Codable, Sendable {
       public let event: String            // "transcript.created" or "test"
       public let transcriptID: Int64?
       public let createdAt: Date
       public let sourceType: String       // "dictation" or "imported_file"
       public let modelID: String
       public let durationSeconds: Double?
       public let text: String
       public let speakerMap: [String: String]?   // if includeSpeakerMap
       public let segments: [WebhookSegment]?     // if includeSegments
   }

   public struct WebhookSegment: Codable, Sendable {
       public let start: Double
       public let end: Double
       public let text: String
       public let speaker: String?
   }
   ```
2. Create `WebhookService`:
   - `func deliver(transcript: TranscriptRecord, config: WebhookConfiguration) async`
     - Builds `WebhookPayload` from transcript + config options
     - Encodes to JSON with `.iso8601` date strategy
     - Signs the body with HMAC-SHA256 if `hmacSecret` is non-empty:
       - `X-Waffle-Signature: sha256={hex(HMAC-SHA256(secret, body))}`
     - Adds headers: `Content-Type: application/json`, `User-Agent: Waffle/{appVersion}`, `X-Waffle-Event: transcript.created`
     - POST to `endpointURL`, 10-second timeout
     - Retry: up to 3 attempts with exponential backoff (2s, 4s, 8s) on network errors or 5xx status codes
     - On 4xx: do NOT retry (misconfiguration, not a transient error)
   - `func sendTestPayload(config: WebhookConfiguration) async throws -> Int` — returns HTTP status code

3. In `DictationController`, after `saveTranscriptToHistory` succeeds:
   - If `WebhookConfiguration.load().isEnabled` and `endpointURL` is non-empty:
     - Fire: `Task { await webhookService.deliver(transcript: savedRecord, config: config) }`
     - Fire-and-forget. Webhook delivery must never block or crash the dictation flow.

4. HMAC-SHA256 implementation using `CryptoKit`:
   ```swift
   import CryptoKit
   let key = SymmetricKey(data: Data(secret.utf8))
   let mac = HMAC<SHA256>.authenticationCode(for: body, using: key)
   let hexSignature = mac.map { String(format: "%02x", $0) }.joined()
   ```

**Notes:**
- `CryptoKit` is a system framework on macOS 14 — no new Package.swift dependencies.
- Retry policy: only for network errors (`URLError`) and 5xx HTTP status codes. Never retry 4xx.
- Delivery must be fire-and-forget. Use `Task.detached(priority: .background)` to avoid holding the main actor.
- Total retry duration for 3 attempts: up to 14 seconds. This is acceptable for a background webhook.
- Signature header follows the GitHub webhook signature format: `sha256={hex}`.
- `User-Agent` includes app version for webhook endpoint debugging.

**Tests:** Test HMAC-SHA256 signature generation for a known input/key pair. Test that retry fires on 500 status and not on 400. Test that `deliver` does not throw (all errors are absorbed). Test `sendTestPayload` returns the HTTP status code.

**Acceptance:** After each successful dictation, a signed JSON POST is sent to the configured endpoint. Network failures are retried up to 3 times. 4xx errors are not retried. All delivery is fire-and-forget.

### Task C2: iCloud Backup Service
**Target:** `Sources/WaffleCore/iCloudBackupService.swift` (new)

Auto-backup new transcripts as `.waffle` files to the iCloud Drive container.

1. Create `iCloudBackupService`:
   ```swift
   public final class iCloudBackupService: Sendable {
       private let containerIdentifier: String
       private let archiver: TranscriptArchiver
       private let fileManager: FileManager

       public init(
           containerIdentifier: String = "iCloud.com.waffle.app",
           archiver: TranscriptArchiver = TranscriptArchiver(),
           fileManager: FileManager = .default
       ) { ... }

       public var isAvailable: Bool {
           containerURL != nil
       }

       public var containerURL: URL? {
           fileManager.url(forUbiquityContainerIdentifier: containerIdentifier)?
               .appendingPathComponent("Documents/Transcripts", isDirectory: true)
       }

       public func backup(transcript: TranscriptRecord, actions: [TranscriptActionRecord]) throws {
           guard let container = containerURL else { return }  // silently skip if unavailable
           try fileManager.createDirectory(at: container, withIntermediateDirectories: true)
           let archive = WaffleArchive(
               version: TranscriptArchiver.currentArchiveVersion,
               exportedAt: Date(),
               appVersion: /* bundle version */ "unknown",
               transcripts: [ArchivedTranscript(record: transcript, actions: actions)]
           )
           let filename = backupFilename(for: transcript)
           let destination = container.appendingPathComponent(filename)
           try archiver.write(archive, to: destination)
       }

       public func listBackups() throws -> [URL] {
           guard let container = containerURL else { return [] }
           let files = try fileManager.contentsOfDirectory(
               at: container,
               includingPropertiesForKeys: [.contentModificationDateKey],
               options: .skipsHiddenFiles
           )
           return files
               .filter { $0.pathExtension == "waffle" }
               .sorted { ... by contentModificationDate descending ... }
       }

       public func deleteBackup(at url: URL) throws {
           try fileManager.removeItem(at: url)
       }
   }
   ```
   - `backupFilename(for:)` → `{ISO8601date}_{modelID}-transcript.waffle`
     - Replace `:` and spaces with `-` for filename safety
   - `backup(transcript:actions:)` silently returns (no throw) when `containerURL` is nil — iCloud not available

2. Add `iCloudBackupEnabled` preference (Bool, default: false) to UserDefaults

3. In `DictationController`, after `saveTranscriptToHistory` succeeds, if `iCloudBackupEnabled` is true:

   ```swift
   Task.detached(priority: .utility) {
       guard let transcriptID = savedRecord?.id, let store = transcriptStore else { return }
       let actions = (try? store.fetchActions(forTranscriptID: transcriptID)) ?? []
       try? iCloudBackupService.backup(transcript: savedRecord, actions: actions)
   }
   ```
   - Fire-and-forget. Never blocks dictation flow.

4. Add `iCloudBackupService` to `FileTranscriptionService` workflow as well (backup on file import when enabled)

**Notes:**
- `url(forUbiquityContainerIdentifier:)` blocks until iCloud metadata downloads. Call from a background thread (already in a `Task.detached`).
- The `Documents/Transcripts/` subdirectory inside the container is user-visible in Finder under `iCloud Drive > Waffle > Transcripts`.
- Backup filename uses ISO-8601 date (no time, for human readability): `{YYYY-MM-DD}_{modelID}-transcript.waffle`.
- If a file with the same name already exists, overwrite it (same transcript re-exported). This is idempotent.
- `backup` swallows errors silently. Failures appear in the iCloud Drive status, not as app errors.

## Notes for Codex

- **Swift version:** 6.0+ strict concurrency. All new types must be `Sendable`.
- **Test framework:** Swift Testing (`import Testing`, `@Test`, `#expect`) — NOT XCTest.
- **No new Swift packages.** `AppIntents` and `CryptoKit` are system frameworks on macOS 14. `iCloud` uses `Foundation.FileManager`.
- **AppIntents framework:** `import AppIntents`. Available on macOS 13+. No Package.swift changes needed.
- **`DictationIntentBridge`:** `@MainActor` singleton, weak reference to `DictationController`. The bridge enables App Intents (which run off-main-actor) to communicate with the main-actor controller safely.
- **`TranscriptIntentBridge`:** `@unchecked Sendable` (not `@MainActor`) since `TranscriptStore` is itself `@unchecked Sendable` and `fetchAll` is a synchronous DB call.
- **`IntentError`:** Use `IntentError.custom("message")` to surface user-facing errors from `AppIntent.perform()`.
- **`AppShortcutsProvider`:** Register in `WaffleApp.swift` or `AppDelegate.swift`. Must be a top-level struct (not nested).
- **HMAC-SHA256:** `import CryptoKit`. `SymmetricKey(data: Data(secret.utf8))`. `HMAC<SHA256>.authenticationCode(for: body, using: key)`. Format as `sha256={hexString}`.
- **Webhook retry:** Use `Task.sleep(for: .seconds(delay))` between retries inside `deliver(...)`. Delays: 2s, 4s, 8s. Total max wait ≈ 14s.
- **`WebhookConfiguration` UserDefaults key:** `webhookConfiguration`. Encode/decode with `JSONEncoder/JSONDecoder`.
- **`WebhookDeliveryLog` UserDefaults key:** `webhookDeliveryLog`. Trim to last 10 entries after each append.
- **URL validation:** Use `URL(string: urlString)` and check `scheme == "https"`. Empty string is treated as disabled.
- **iCloud container identifier:** `"iCloud.com.waffle.app"` (adjust to actual bundle ID). Check with `FileManager.default.url(forUbiquityContainerIdentifier:)`.
- **`url(forUbiquityContainerIdentifier:)` is synchronous but may stall briefly.** Always call from a background thread (inside `Task.detached`). Never call on the main thread.
- **iCloud backup file naming:** Sanitize the ISO-8601 date by replacing `:` with `-` (colons are invalid in macOS filenames from the Finder perspective, even though they work in POSIX paths). Use `yyyy-MM-dd` only (no time component) for human-readable filenames.
- **`NSWorkspace.shared.activateFileViewerSelecting`:** Available on macOS 14 without sandboxing issues for paths within the user's home directory.
- **Settings tab changes:** Add a "Back up to iCloud Drive" toggle to `GeneralSettingsView`. Add a "Webhooks" section to `AISettingsView`. No new tabs needed.
- **`iCloudBackupEnabled` UserDefaults key:** `iCloudBackupEnabled`, Bool, default false.
- **`webhookConfiguration` and `WebhookConfiguration`:** Treat `hmacSecret` as sensitive. Do not log it. The `WebhookPayload` JSON should NOT include the secret.
- **`FileTranscriptionService` backup integration:** `FileTranscriptionService` returns `FileTranscriptionResult`. The backup call belongs in `DictationController` (which already handles both dictation and file import flows). Do not modify `FileTranscriptionService` directly.
- **Test the iCloud path with mock FileManager:** `iCloudBackupService` accepts a `fileManager` parameter in its initializer for testing. Use a mock `FileManager` (or a temporary directory) in tests to avoid requiring a real iCloud connection.
- **App Intents and `@MainActor`:** `StartDictationIntent.perform()` and `StopDictationIntent.perform()` must be marked `@MainActor` to safely access `DictationController`. `GetLastTranscriptIntent` and `SearchTranscriptsIntent` access `TranscriptStore` (which is thread-safe via GRDB) and do not need `@MainActor`.
