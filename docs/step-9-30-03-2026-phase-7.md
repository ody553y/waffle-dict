### Task C2: Speaker Profile Storage in Swift
**Target:** `Sources/ScreamerCore/TranscriptStore.swift`, `Sources/ScreamerCore/SpeakerProfileStore.swift` (new), `Sources/ScreamerCore/WorkerClient.swift`

Store speaker embeddings per transcript and match them to persistent cross-session speaker profiles.

1. Update `FileTranscriptionResponsePayload` in `WorkerClient.swift`:
   - Add `speakerEmbeddings: [String: [Float]?]?` — maps `SPEAKER_XX` → embedding vector (or nil for insufficient audio)
2. Create `SpeakerProfile` and `SpeakerProfileStore`:
   ```swift
   public struct SpeakerProfile: Codable, Identifiable, Sendable {
       public var id: UUID
       public var displayName: String   // "Alice", "Bob"
       public var createdAt: Date
       public var lastSeenAt: Date
       public var averageEmbedding: [Float]   // running average of matched embeddings
       public var transcriptCount: Int
   }
   ```
   - GRDB table `speaker_profiles`:
     - `id TEXT PRIMARY KEY` (UUID string)
     - `displayName TEXT NOT NULL`
     - `createdAt REAL NOT NULL` (Unix timestamp)
     - `lastSeenAt REAL NOT NULL`
     - `averageEmbedding BLOB NOT NULL` (packed Float32 array)
     - `transcriptCount INTEGER NOT NULL DEFAULT 1`
   - Methods:
     - `fetchAll() throws -> [SpeakerProfile]`
     - `save(_ profile: SpeakerProfile) throws`
     - `delete(id: UUID) throws`
     - `findMatch(for embedding: [Float], threshold: Float) throws -> SpeakerProfile?`
     - `updateAverageEmbedding(id: UUID, newEmbedding: [Float]) throws`
3. Create `transcript_speaker_embeddings` table in `TranscriptStore`:
   - Columns: `transcriptID INTEGER NOT NULL REFERENCES transcripts(id) ON DELETE CASCADE`, `speakerLabel TEXT NOT NULL`, `embedding BLOB NOT NULL`
   - Primary key: `(transcriptID, speakerLabel)`
   - Methods on `TranscriptStore`:
     - `saveEmbeddings(_ embeddings: [String: [Float]], transcriptID: Int64) throws`
     - `fetchEmbeddings(transcriptID: Int64) throws -> [String: [Float]]`
4. Cosine similarity:
   ```swift
   public func cosineSimilarity(_ a: [Float], _ b: [Float]) -> Float {
       guard a.count == b.count, !a.isEmpty else { return 0 }
       let dot = zip(a, b).reduce(0.0 as Float) { $0 + $1.0 * $1.1 }
       let normA = sqrt(a.reduce(0.0 as Float) { $0 + $1 * $1 })
       let normB = sqrt(b.reduce(0.0 as Float) { $0 + $1 * $1 })
       guard normA > 0, normB > 0 else { return 0 }
       return dot / (normA * normB)
   }
   ```
   - Match threshold: 0.85 (default, exposed as `speakerMatchThreshold` in UserDefaults, range 0.70–0.99)
   - If similarity ≥ threshold → matched profile
   - If no match → create new profile with auto-name `"Speaker N"` (N = `fetchAll().count + 1`)
5. Running average update when a new transcript matches an existing profile:
   - `newAvg[i] = (oldAvg[i] * n + newEmbedding[i]) / (n + 1)` where `n = profile.transcriptCount` before increment
   - Increment `transcriptCount`, update `lastSeenAt`
6. BLOB encoding/decoding for `[Float]`:
   ```swift
   // Encode
   var array = embedding
   let data = Data(bytes: &array, count: array.count * MemoryLayout<Float>.size)
   // Decode
   let floats = data.withUnsafeBytes { Array($0.bindMemory(to: Float.self)) }
   ```

**Notes:**
- `transcript_speaker_embeddings` enables future re-matching when the user adjusts the threshold or merges profiles.
- Matching and profile creation happen in `DictationController` after saving the transcript + embeddings, as a background task (same fire-and-forget pattern as auto-summarize).
- The `SpeakerProfileStore` uses the same GRDB `DatabaseQueue` as `TranscriptStore` — pass it as a dependency.

**Target:** `Sources/ScreamerCore/TranscriptArchiver.swift` (new)

Define and implement a portable archive format for sharing transcripts with their AI action history.

1. Create `ScreamerArchive` and `TranscriptArchiver` in `ScreamerCore`:
   ```swift
   public struct ScreamerArchive: Codable, Sendable {
       public let version: Int          // 1
       public let exportedAt: Date
       public let appVersion: String    // bundle short version string
       public var transcripts: [ArchivedTranscript]
   }

   public struct ArchivedTranscript: Codable, Sendable {
       public var record: TranscriptRecord
       public var actions: [TranscriptActionRecord]
   }
   ```
2. File format:
   - JSON encoded with `.prettyPrinted` + `.sortedKeys` for deterministic output
   - File extension: `.scream`
   - UTF-8 encoding
   - MIME type: `application/json` (no custom UTI required in Phase 7)
3. `TranscriptArchiver` methods:
   - `export(transcripts: [TranscriptRecord], store: TranscriptStore) throws -> Data`
     - Fetches all `TranscriptActionRecord`s for each transcript via `TranscriptStore.fetchActions(forTranscriptID:)`
     - Builds `ScreamerArchive`, JSON-encodes
   - `import(from data: Data) throws -> ScreamerArchive`
     - Decodes JSON
     - Validates: if `version > 1` throw `ArchiveError.unsupportedVersion(version)`
   - `write(_ archive: ScreamerArchive, to url: URL) throws`
   - `read(from url: URL) throws -> ScreamerArchive`
4. Error type:
   ```swift
   public enum ArchiveError: Error, LocalizedError {
       case unsupportedVersion(Int)
       case invalidData(String)
   }
   ```

**Notes:**
- `.scream` is memorable and distinct. It won't collide with any standard macOS extension.
- Pretty-printed JSON makes archives human-readable and diff-friendly.
- `id` fields on records will be set to nil before import so the DB assigns fresh IDs.
- `appVersion` is informational only. No compatibility logic based on it in v1.
- No compression in v1. Text-heavy archives compress well with gzip if needed in a future version.

**Tests:** Test round-trip: `export` → `write` → `read` → `import` produces identical `TranscriptRecord`s and `TranscriptActionRecord`s. Test `unsupportedVersion` error when `version > 1`. Test single and multi-transcript archives. Test empty actions array is handled correctly.

### Task D3: Import Archive
**Target:** `Sources/ScreamerApp/TranscriptHistoryView.swift`, `Sources/ScreamerApp/ScreamerApp.swift`

Allow importing `.scream` archives to restore transcripts and their AI action history.

1. Add "Import Archive…" button in the `TranscriptHistoryView` toolbar:
   - Opens `NSOpenPanel` filtered to files with extension `scream` or `json`
   - Reads file data, calls `TranscriptArchiver.import(from:)`
2. Add drag-and-drop support to `TranscriptHistoryView`:
   - Drop target accepts files with extension `scream` or `json`
   - `.onDrop(of: [UTType.fileURL], isTargeted: $isDragTargeted)` with visual highlight when `isDragTargeted`
   - On drop: load file URL from `NSItemProvider`, read data, import
3. Add `File > Import Transcript Archive…` menu item in `ScreamerApp.swift` (uses the same action as the toolbar button)
4. Import workflow (all DB writes in a single transaction):
   ```swift
   try dbQueue.write { db in
       for archivedTranscript in archive.transcripts {
           guard !isDuplicate(archivedTranscript.record, db: db) else { skipped += 1; continue }
           var record = archivedTranscript.record
           record.id = nil  // force DB to assign new ID
           try record.insert(db)
           let newID = record.id!
           for var action in archivedTranscript.actions {
               action.id = nil
               action.transcriptID = newID
               try action.insert(db)
           }
           imported += 1
       }
   }
   ```
5. Duplicate detection heuristic:
   - A transcript is a duplicate if an existing record matches on `(createdAt, modelID, text.prefix(200))`
   - Duplicates are skipped; not an error
6. After import: show summary alert: "Imported N transcripts with M AI actions. Skipped X duplicates."
   - Reload the transcript list
   - Scroll to and highlight the first imported transcript
7. Error cases: invalid JSON, unsupported version, corrupt data → show an alert with the localized error message

**Notes:**
- Import is fully atomic via GRDB transaction. On any error, nothing is persisted.
- `text.prefix(200)` for duplicate detection is a fast heuristic. Using the full text risks false negatives from minor whitespace differences; using too little risks false positives.
- Drag-and-drop: use `NSItemProvider.loadItem(forTypeIdentifier: UTType.fileURL.identifier)` to get the file URL.
- The same `.scream` archive can be safely imported multiple times — duplicates will be skipped each time.

**Tests:** Test that import persists `TranscriptRecord` with a new DB-assigned `id`. Test that `TranscriptActionRecord`s are saved with the correct new `transcriptID` FK. Test duplicate detection skips re-import of the same transcript. Test transactional rollback: if any insert fails, no records are committed.

**Acceptance:** Users can import `.scream` archives via toolbar, File menu, or drag-and-drop. Imported transcripts and their AI action history appear in the history view. Duplicates are skipped with a count in the summary. Import is atomic.

## Notes for Codex

- **Swift version:** 6.0+ strict concurrency throughout. New types must be `Sendable` where appropriate.
- **Test framework:** Swift Testing (`import Testing`, `@Test`, `#expect`) — NOT XCTest.
- **Python test framework:** `unittest` — match existing worker test patterns.
- **No new Swift packages** in Phase 7. All work uses `AVFoundation` (system framework, already available), `GRDB` (existing), `Accelerate` (optional, system framework, for cosine similarity if needed), and standard library.
- **No new Python dependencies.** The pyannote speaker embedding model is already loaded during diarization — re-use it.
- **Database migrations:** Append after existing migrations in `TranscriptStore`. Phase 7 adds: `audioFilePath TEXT` column on `transcripts`, new table `speaker_profiles`, new table `transcript_speaker_embeddings`.
- **`PromptTemplateStore`:** Stored in `UserDefaults` as a JSON array under key `promptTemplates`. No DB table.
- **Embedding BLOB storage:** Pack `[Float]` → `Data` via `Data(bytes: &array, count: array.count * MemoryLayout<Float>.size)`. Unpack via `data.withUnsafeBytes { Array($0.bindMemory(to: Float.self)) }`.
- **`AVAudioPlayer`:** Not `Sendable` — confine to `@MainActor`. Import `AVFoundation`. Use `AVAudioPlayer` (not `AVPlayer`) for local WAV files.
- **Timer for playback:** `Timer.publish(every: 0.1, on: .main, in: .common).autoconnect()` in SwiftUI. Cancel on view disappear or when `isPlaying` becomes false.
- **Auto-summarize:** Fire-and-forget `Task`. Catch all errors. Log at `.debug` with `os_log`. Never surface to the user.
- **Template seeding:** Check `UserDefaults` on `PromptTemplateStore.load()`. If the key is absent (nil), seed defaults and call `save`. If the key is present but the array is empty, do NOT re-seed (user may have intentionally deleted all templates).
- **`ScreamerArchive` version:** Hardcode `version: 1`. On import: throw `ArchiveError.unsupportedVersion(version)` if `version > 1`.
- **Import transaction:** Wrap all inserts in a single `DatabaseQueue.write { db in ... }` block. On any error, the transaction rolls back automatically.
- **Duplicate detection key:** `(createdAt, modelID, text.prefix(200))` — use `DateComponents` comparison at second precision for `createdAt` (avoid floating-point issues).
- **Drag-and-drop import:** `.onDrop(of: [UTType.fileURL], isTargeted: $isDragTargeted)`. Load URL from provider with `loadItem(forTypeIdentifier: UTType.fileURL.identifier)`, then cast to `URL`.
- **Speaker profile auto-naming:** Query `SpeakerProfileStore.fetchAll()`, find the highest integer N in names matching `"Speaker N"`, use N+1. Default to `"Speaker 1"` if no profiles exist.
- **`SpeakerProfileStore`:** Uses the same GRDB `DatabaseQueue` as `TranscriptStore`. Pass the queue as a constructor dependency — do not open a separate SQLite connection.
- **File menu item:** Add `commands { CommandGroup(replacing: .newItem) { ... } }` in `ScreamerApp.swift` or use an existing command group. The "Import Archive…" item goes in the `CommandGroup(after: .newItem)` position.
- **Settings tab order:** General, Models, AI, Keyboard (unchanged). Speaker Profiles section goes inside the AI tab, below LM Studio settings. Prompt Templates section also in the AI tab.
- **Audio file path handling:** Always check `FileManager.default.fileExists(atPath:)` before attempting playback. Show "Audio file not found" gracefully rather than throwing.
